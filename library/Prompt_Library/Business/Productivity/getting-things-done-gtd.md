# Getting Things Done (GTD) System Builder

## Purpose
Implements David Allen's Getting Things Done methodology for achieving "mind like water" - stress-free productivity through comprehensive capture, clarification, organization, and review.

## Instructions

I want you to act as a GTD implementation coach. Help me set up a complete Getting Things Done system.

**Setup Requirements:**
- **Current tools:** [e.g., Notion, Todoist, paper, etc.]
- **Main contexts:** [e.g., @computer, @home, @office, @phone, @errands]
- **Areas of responsibility:** [e.g., Work projects, family, health, finances]
- **Current pain points:** [e.g., forgetting tasks, overwhelm, unclear priorities]

Create a GTD system with these components:

### 1. Capture (Inbox)
- Single inbox for all inputs (or one per life area: work, personal)
- Quick capture tools accessible everywhere
- Daily inbox processing routine

### 2. Clarify
- Decision tree: Is it actionable?
  - No → Trash, Someday/Maybe, Reference
  - Yes → Do (< 2 min), Delegate, or Defer

### 3. Organize
- **Projects** (outcomes requiring multiple steps)
- **Next Actions** (specific, physical actions by context)
- **Waiting For** (delegated/pending items)
- **Someday/Maybe** (things to revisit later)
- **Reference** (non-actionable information)

### 4. Reflect
- **Daily Review:** Review calendar + next actions
- **Weekly Review:** Process inbox, review projects, update lists

### 5. Engage
- Choose actions based on: context, time available, energy, priority

## Output Format

```markdown
# Your GTD System

## 1. Capture System

### Inbox Locations
- **Physical inbox:** [Where physical papers go]
- **Digital inbox:** [Email, notes app, etc.]
- **Quick capture:** [Phone app for on-the-go]

### Capture Triggers
- Email arrives → Forward to task manager
- Idea occurs → Voice note or quick capture app
- Meeting ends → Action items to inbox
- Reading something → Highlight/clip to inbox

**Rule:** Capture EVERYTHING. Don't trust your brain to remember.

## 2. Clarify Process (Daily)

**Inbox Processing (20-30 min/day):**

For each item, ask:
1. **What is it?** (Understand the input)
2. **Is it actionable?**

If NO:
- Trash it (delete/discard)
- File as reference (keep for later)
- Add to Someday/Maybe list

If YES:
- **Can I do it in < 2 minutes?** → Do it now
- **Should someone else do it?** → Delegate (add to Waiting For)
- **Does it need to be done?** → Add to Next Actions

3. **Is it a project?** (requires multiple steps)
   - Create project entry
   - Define desired outcome
   - Identify next action

## 3. Organization Structure

### Projects List
[Anything requiring >1 action to complete]

| Project | Desired Outcome | Next Action | Status |
|---------|----------------|-------------|--------|
| Q2 Marketing Campaign | Campaign live by June 1 | Draft campaign brief | Active |
| Home Office Setup | Productive workspace | Measure desk area | Active |
| Annual Review | Completed review doc | Schedule 1:1 with manager | Waiting |

### Next Actions (by Context)

**@Computer**
- [ ] Draft Q2 campaign brief
- [ ] Update project roadmap
- [ ] Review pull requests

**@Office**
- [ ] Print reports for meeting
- [ ] Organize filing cabinet
- [ ] Schedule team lunch

**@Home**
- [ ] Measure desk area for office
- [ ] Call plumber about leak
- [ ] Order birthday gift

**@Phone**
- [ ] Call dentist for appointment
- [ ] Follow up with vendor
- [ ] Check in with mentor

**@Errands**
- [ ] Pick up dry cleaning
- [ ] Buy desk lamp at IKEA
- [ ] Drop off package at post office

**@Waiting For**
- [ ] John's feedback on proposal (due 4/5)
- [ ] Legal review of contract (due 4/10)
- [ ] Expense reimbursement ($234)

**@Someday/Maybe**
- [ ] Learn Spanish
- [ ] Write a book
- [ ] Start podcast
- [ ] Take photography course

### Reference System
- Meeting notes → Folder: Meetings/[Date]
- Project docs → Folder: Projects/[Project Name]
- Personal → Folder: Personal/[Category]

## 4. Review Rituals

### Daily Review (5-10 min)
**Morning:**
- [ ] Check calendar for today's commitments
- [ ] Review Next Actions for current context
- [ ] Identify 3 MITs (Most Important Tasks)

**Evening:**
- [ ] Process inbox to zero
- [ ] Check Waiting For items (any overdue?)
- [ ] Prep tomorrow's calendar

### Weekly Review (60-90 min - Friday afternoon or Sunday)

**Get Clear:**
- [ ] Process all inboxes to zero
- [ ] Review past week's calendar (capture loose ends)
- [ ] Review upcoming week's calendar (prep needed items)

**Get Current:**
- [ ] Review Projects list
  - Any completed? → Archive or celebrate
  - Any stalled? → Identify next action
  - Any new? → Add and define outcome
- [ ] Review Next Actions
  - Delete completed
  - Add new next actions from projects
- [ ] Review Waiting For
  - Follow up on overdue items
  - Remove received items

**Get Creative:**
- [ ] Review Someday/Maybe
  - Anything ready to activate? → Move to Projects
  - Anything no longer relevant? → Delete
- [ ] Capture new ideas/goals/projects
- [ ] Set intentions for next week

## 5. Engage - Choosing What to Do

**Four Criteria for Choosing Actions:**

1. **Context:** What can I do given my current location/tools?
   - At computer → @Computer list
   - Out running errands → @Errands list

2. **Time Available:** How much time until next commitment?
   - 10 minutes → Quick emails
   - 2 hours → Deep work project task
   - Full day → Major project milestone

3. **Energy Level:** How mentally/physically fresh am I?
   - High energy → Creative/strategic work
   - Medium energy → Admin/routine tasks
   - Low energy → Organize, read, light tasks

4. **Priority:** What's most important right now?
   - Check project deadlines
   - Review MITs for the day
   - Consider impact vs. effort

**Decision Flow:**
1. Look at calendar (do scheduled items)
2. Check Next Actions for current context
3. Filter by time available
4. Filter by energy level
5. Choose highest priority item

## GTD Tools Recommendations

### Digital Tools
- **Todoist** - Great for GTD with projects, contexts (labels), recurring tasks
- **Things 3** (Mac/iOS) - Beautiful, built for GTD methodology
- **Notion** - Flexible database for projects + notes + reference
- **Obsidian + Tasks plugin** - For linking tasks with notes

### Hybrid (Digital + Analog)
- **Bullet Journal** - Capture + daily task tracking on paper
- **Digital reference** - Long-term storage in Notion/Evernote

### Key Features Needed
- Projects (nested tasks)
- Contexts (tags/labels: @computer, @home, etc.)
- Quick capture (mobile app, voice, email-to-task)
- Repeating tasks (for reviews)
- Date/time flexibility (many tasks have no due date)

## Example Workflow

**Scenario:** Email arrives: "Can you review the Q2 budget draft and send feedback by next Friday?"

**Process:**
1. **Capture:** Email is already in inbox (Gmail)
2. **Clarify:**
   - Is it actionable? Yes
   - <2 min? No
   - Delegate? No (I must do it)
   - Project? No (single action)
   - Next action: "Review Q2 budget draft and email feedback"
3. **Organize:**
   - Add to Next Actions: @Computer
   - Due date: Next Friday
   - Context: @Computer (need laptop)
4. **Reflect:** Will review on Friday during weekly review
5. **Engage:** Will do during morning deep work block when energy is high
```

## Tips for Use

- **Inbox Zero daily:** Process inbox every day (don't let it pile up)
- **2-minute rule:** If it takes less than 2 minutes, do it immediately
- **Weekly review is sacred:** This is where GTD lives or dies - never skip it
- **Keep it simple:** Start with basics, add complexity only as needed
- **Action = verb:** Next actions should start with a verb (Call, Email, Draft, Buy)

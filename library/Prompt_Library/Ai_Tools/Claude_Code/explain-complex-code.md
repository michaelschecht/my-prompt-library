Write a plain-English explanation of tricky code for a new team member.

## Prompt

```
Read [FILE or FUNCTION]. Write a plain-English explanation suitable for a new team member. Cover: what problem it solves, how it works conceptually, why it was implemented this way (not just what), and any gotchas. Use analogies if helpful.
```

## Usage Notes

- Replace `[FILE or FUNCTION]` with the code to explain
- Focus on why over what — the code already shows what
- Analogies are encouraged for abstract or algorithmic code
- Good for onboarding docs, wiki pages, or inline comments

Analyze existing architecture and produce an opinionated improvement report.

## Prompt

```
Read the codebase structure, key modules, and data flows. Produce an architecture review with:
1) Current architecture diagram (ASCII or described)
2) What is working well
3) Pain points and risks
4) Specific, prioritized improvement recommendations

Be opinionated.
```

## Usage Notes

- Read broadly across the codebase before writing the review
- The diagram section gives a shared mental model
- Improvements should be prioritized by impact, not comprehensiveness
- "Be opinionated" produces actionable output rather than hedged non-answers

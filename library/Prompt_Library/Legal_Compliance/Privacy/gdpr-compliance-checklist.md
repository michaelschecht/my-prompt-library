# GDPR Compliance Checklist Generator

Create comprehensive GDPR compliance checklists for your organization.

## Prompt

```
Generate a GDPR compliance checklist for:

**Organization Type:** [e.g., SaaS company, ecommerce, healthcare]
**Data Processing Activities:** [Describe what personal data you collect/process]
**User Base:** [EU users percentage, total users]

**Create checklist covering:**

## 1. Data Protection Principles
- [ ] Lawfulness, fairness, transparency
- [ ] Purpose limitation
- [ ] Data minimization
- [ ] Accuracy
- [ ] Storage limitation
- [ ] Integrity and confidentiality
- [ ] Accountability

## 2. Legal Basis for Processing
- [ ] Consent obtained where required
- [ ] Legitimate interest assessments conducted
- [ ] Contract necessity documented
- [ ] Legal obligation compliance
- [ ] Vital interests protection
- [ ] Public task execution

## 3. Data Subject Rights
- [ ] Right to access (data export feature)
- [ ] Right to rectification (edit profile)
- [ ] Right to erasure ("right to be forgotten")
- [ ] Right to restrict processing
- [ ] Right to data portability
- [ ] Right to object
- [ ] Rights related to automated decision-making

## 4. Technical & Organizational Measures
- [ ] Encryption at rest and in transit
- [ ] Access controls and authentication
- [ ] Audit logs and monitoring
- [ ] Data backup and recovery
- [ ] Incident response plan
- [ ] Privacy by design implemented
- [ ] Data protection impact assessments (DPIAs)

## 5. Documentation
- [ ] Records of processing activities (ROPA)
- [ ] Privacy policy published
- [ ] Cookie policy (if applicable)
- [ ] Data processing agreements (DPAs) with vendors
- [ ] DPIA documentation
- [ ] Consent records
- [ ] Data breach register

## 6. Roles & Responsibilities
- [ ] Data Protection Officer (DPO) appointed (if required)
- [ ] EU Representative designated (if needed)
- [ ] Staff training on GDPR
- [ ] Processor contracts in place

## 7. Breach Response
- [ ] Breach detection procedures
- [ ] 72-hour notification process to supervisory authority
- [ ] Individual notification procedures
- [ ] Breach documentation process

For each item, provide:
- Implementation steps
- Responsible party
- Timeline
- Tools/resources needed
```

## Example Outputs

**For SaaS Company:**
"Your SaaS platform requires: consent management platform (OneTrust, Cookiebot), data export API, DPIA for AI features, DPO appointment (>250 employees or core activity is systematic monitoring)."

**For Ecommerce:**
"Cookie consent banner required, payment processor DPA, customer data export within 30 days, right-to-be-forgotten workflow for order history."

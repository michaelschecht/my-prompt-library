# 2. Cyber Security Prompt Library

This document contains categorized prompts for building Cyber Security documentation, standards, policies, and controls.

1. Develop a comprehensive cybersecurity policy framework that includes acceptable use, remote access, and data protection.
2. Create a security awareness training program outline, including annual training modules and phishing simulation campaigns.
3. Document a network security standard covering firewall rules, segmentation, and intrusion prevention system (IPS) configurations.
4. Write a security incident classification matrix defining severity levels (P1-P4) with corresponding response and escalation criteria.
5. Establish a standard for endpoint detection and response (EDR) deployment, including coverage requirements and alert triage workflows.
6. Define a secure configuration baseline for network devices (routers, switches, firewalls) aligned with CIS benchmarks.
7. Create a third-party risk management (TPRM) process document, including vendor security assessment questionnaires and due diligence steps.
8. Document a data loss prevention (DLP) strategy covering data at rest, in motion, and in use, with specific policy rules.
9. Develop a cybersecurity incident communication plan for internal and external stakeholders during a breach.
10. Write a penetration testing standard outlining scope, methodology (e.g., OWASP, PTES), remediation validation, and reporting requirements.
11. Define a security exception management process for deviations from policy, including risk acceptance and temporary waivers.
12. Create a secure software development lifecycle (SSDLC) policy integrating security gates at each phase (requirements, design, code, test, deploy).
13. Document a vulnerability disclosure program (VDP) for external researchers, including reporting channels and response SLAs.
14. Establish a cybersecurity metrics dashboard with key performance indicators (KPIs) and key risk indicators (KRIs) for board reporting.
15. Write a standard for cloud workload protection platform (CWPP) deployment across hybrid and multi-cloud environments.
16. Create a security tooling roadmap covering EDR, SIEM, SOAR, and vulnerability management evolution over 3-5 years.
17. Define an identity threat detection and response (ITDR) strategy focused on detecting identity-based attacks.
18. Document a security architecture review process for new projects and major changes.
19. Develop a ransomware readiness assessment checklist covering backups, segmentation, and offline recovery capabilities.
20. Write a cybersecurity insurance application support document listing required controls and evidence for underwriters.

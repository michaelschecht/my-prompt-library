# 10. Incident Response Prompt Library

This document contains categorized prompts for building Incident Response documentation, playbooks, and standards.

1. Create an incident response policy document defining roles (IRT), authority, and escalation paths.
2. Develop a security incident classification matrix with severity levels (P1-P4) and corresponding response procedures.
3. Write a ransomware incident response playbook covering detection, containment, eradication, and recovery.
4. Define a data breach notification procedure, including legal, regulatory, and customer communication steps.
5. Create a phishing incident response playbook for handling reported and detected phishing emails.
6. Develop a malware outbreak response playbook with steps for isolation, analysis, and remediation.
7. Write a cloud security incident response playbook for incidents in AWS, Azure, or GCP.
8. Define a business email compromise (BEC) incident response playbook.
9. Create a denial-of-service (DoS/DDoS) response playbook with mitigation and communication steps.
10. Develop an insider threat incident response playbook covering investigation and evidence preservation.
11. Write a standard for digital forensics and evidence collection (chain of custody, disk imaging, memory capture).
12. Define a post-incident review (PIR) process template for capturing lessons learned and action items.
13. Create a communication template for internal and external stakeholders during an incident.
14. Develop a tabletop exercise scenario library for testing incident response plans.
15. Write a standard for integrating threat intelligence into incident response (IoC matching, threat feeds).
16. Define a standard for incident tracking and documentation in ticketing systems (e.g., Jira, ServiceNow).
17. Create a standard for engaging law enforcement and external forensics firms during major incidents.
18. Develop a standard for preserving evidence for legal proceedings (e-discovery holds).
19. Write a standard for incident response metrics and reporting (e.g., MTTR, number of incidents, root causes).
20. Define a standard for continuous improvement of incident response capabilities based on post-incident findings.

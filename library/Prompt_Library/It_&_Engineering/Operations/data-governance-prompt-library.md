# 8. Data Governance Prompt Library

This document contains categorized prompts for building Data Governance documentation, standards, and policies.

1. Create a data governance policy document defining principles, roles (data owners, stewards), and decision rights.
2. Develop a data classification standard with levels (public, internal, confidential, restricted) and handling requirements.
3. Document a data quality framework including dimensions (accuracy, completeness, timeliness) and measurement processes.
4. Write a data retention and disposal policy aligned with legal, regulatory, and business requirements.
5. Define a data sharing agreement template for internal and external data exchanges.
6. Create a standard for master data management (MDM) for critical entities like customer, product, and supplier.
7. Develop a data lineage documentation standard to track data flow from source to consumption.
8. Write a standard for metadata management, including business glossary definitions and technical metadata.
9. Define a data access control standard based on roles and data classification (e.g., row-level security, dynamic masking).
10. Create a standard for data privacy impact assessments (DPIA) for new projects involving personal data.
11. Develop a standard for anonymization and pseudonymization techniques for sensitive data.
12. Write a standard for data catalog tooling and curation (e.g., Collibra, Alation) with stewardship workflows.
13. Define a data incident response procedure for data breaches or unauthorized access.
14. Create a standard for data archiving and retrieval processes for historical data.
15. Develop a standard for data integration and ETL/ELT governance, including validation and monitoring.
16. Write a standard for data sharing with third parties, including contracts, security reviews, and audits.
17. Define a data governance metrics framework (e.g., data quality scores, policy compliance rates).
18. Create a standard for data lake governance, including zone definitions (raw, curated, consumption) and access controls.
19. Develop a standard for data science and AI model governance, including model validation and bias testing.
20. Write a standard for data ethics and responsible AI principles.

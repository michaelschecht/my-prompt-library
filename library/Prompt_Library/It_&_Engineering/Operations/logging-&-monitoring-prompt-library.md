# 9. Logging & Monitoring Prompt Library

This document contains categorized prompts for building Logging and Monitoring documentation, standards, and strategies.

1. Create a comprehensive logging strategy document covering log sources, collection methods, and centralized storage.
2. Develop a standard for log format (e.g., CEF, JSON) and mandatory fields (timestamp, source, severity, user, event ID).
3. Write a standard for log retention policies per log type, balancing compliance and cost.
4. Define a standard for log integrity and tamper-proofing (e.g., hashing, write-once storage).
5. Create a standard for security information and event management (SIEM) architecture and data ingestion.
6. Develop a standard for monitoring critical infrastructure services (CPU, memory, disk, network) with alert thresholds.
7. Write a standard for application performance monitoring (APM) including transaction tracing and error tracking.
8. Define a standard for synthetic monitoring and user experience monitoring (real user monitoring, RUM).
9. Create a standard for log parsing and field extraction rules for SIEM queries.
10. Develop a standard for monitoring Kubernetes clusters (control plane, nodes, pods, container logs).
11. Write a standard for cloud resource monitoring (AWS CloudWatch, Azure Monitor) including custom metrics and dashboards.
12. Define a standard for network flow monitoring (NetFlow, sFlow) and analysis for anomaly detection.
13. Create a standard for alerting and notification routing based on severity and on-call schedules.
14. Develop a standard for log review and analysis procedures for security and operations teams.
15. Write a standard for log source onboarding and verification process for new applications or systems.
16. Define a standard for monitoring compliance with internal SLAs and external regulatory requirements.
17. Create a standard for log aggregation tooling (e.g., Splunk, ELK, Datadog) configuration and index management.
18. Develop a standard for incident correlation and root cause analysis using monitoring data.
19. Write a standard for monitoring DevOps pipelines (build, test, deploy) with alerts for failures.
20. Define a standard for capacity planning and trend analysis using historical monitoring data.

## Examples

### Example 1: Comprehensive Cleaning Pipeline

**Context:** Preparing a messy customer database for analysis.

**Task:** Remove duplicates, fix dates, handle outliers, and export to Parquet.

**Commands (Prompt):**
```text
Write a Pandas data cleaning pipeline for 'customers.csv'.
1. Drop exact duplicate rows.
2. Convert 'BirthDate' to datetime, coercing errors to NaT.
3. In the 'Income' column, cap outliers above the 99th percentile to the 99th percentile value.
4. Create a new column 'Is_Adult' (boolean) based on 'BirthDate'.
5. Save the result to 'cleaned_customers.parquet'.
```

**Output:**
```python
import pandas as pd
import numpy as np

def clean_customer_data(file_path):
    # Load data
    print("Loading data...")
    df = pd.read_csv(file_path)

    # 1. Drop duplicates
    initial_shape = df.shape
    df = df.drop_duplicates()
    print(f"Dropped {initial_shape[0] - df.shape[0]} duplicate rows.")

    # 2. Convert to datetime
    df['BirthDate'] = pd.to_datetime(df['BirthDate'], errors='coerce')

    # 3. Cap outliers in 'Income'
    if 'Income' in df.columns:
        p99 = df['Income'].quantile(0.99)
        df['Income'] = np.where(df['Income'] > p99, p99, df['Income'])
        print(f"Capped Income outliers at {p99}")

    # 4. Create 'Is_Adult'
    today = pd.Timestamp('today')
    # Calculate age in years accurately handling leap years
    df['Age'] = (today - df['BirthDate']).astype('<m8[Y]')
    df['Is_Adult'] = df['Age'] >= 18

    # 5. Save to Parquet
    output_file = 'cleaned_customers.parquet'
    df.to_parquet(output_file, index=False)
    print(f"Data saved to {output_file}")

    return df

# Execute
cleaned_df = clean_customer_data('customers.csv')
```

## Troubleshooting

### Common Issues

#### Issue 1: `SettingWithCopyWarning`

**Symptoms:**
- Python throws a warning when executing the generated code, often when assigning a new column to a filtered dataframe.

**Cause:**
Pandas is unsure if you are modifying a view or a copy of the original dataframe.

**Solution:**
Ensure the agent generated code using `.loc[]` or explicitly created a copy using `.copy()` after filtering.
```python
# Fix
df.loc[df['Age'] < 18, 'Status'] = 'Minor'
# Or
filtered_df = df[df['Age'] > 0].copy()
```

#### Issue 2: Date Parsing Failures

**Symptoms:**
- `pd.to_datetime()` raises an error or parses dates incorrectly (e.g., swapping months and days).

**Cause:**
Ambiguous date formats in the raw data.

**Solution:**
Modify the generated code to explicitly define the format string.
```python
df['Date'] = pd.to_datetime(df['Date'], format='%d/%m/%Y', errors='coerce')
```

## Best Practices

### Do's ✅

- Use vectorized operations (e.g., `df['A'] + df['B']`) instead of iterating through rows with `iterrows()`.
- Explicitly state how you want to handle errors (e.g., `errors='coerce'` vs `errors='raise'`).
- Always check `.info()` and `.isnull().sum()` before and after running the cleaning script.

### Don'ts ❌

- Don't use `.apply()` for simple math or string operations; vectorized string methods (`df['col'].str.replace()`) are much faster.
- Don't drop missing values without understanding why they are missing; sometimes a missing value represents a distinct category.

## Performance Considerations

### Optimization Tips

1. **Avoid `iterrows()`**: It is notoriously slow in Pandas.
   ```python
   # Slow
   for index, row in df.iterrows(): ...
   # Fast
   df['new_col'] = np.where(df['condition'], value1, value2)
   ```
2. **Categorical Data**: Converting string columns with low cardinality (few unique values) to the `category` datatype drastically reduces memory footprint.

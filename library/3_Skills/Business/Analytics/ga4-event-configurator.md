# GA4 Event Configurator

Creates standardized dataLayer push snippets and Google Tag Manager configurations for consistent Google Analytics 4 event tracking across web and mobile platforms.

## Prerequisites

- **Required Tool/Service:** Google Tag Manager / GA4 Property

### Setup Instructions

1. Have website/app data model ready
2. Identify key user actions to track

## Usage

### Basic Usage

Generate a standard event snippet.

```javascript
ga4-configurator generate --event "generate_lead" --params "currency,value"
```

**What it does:**
1. Validates event name against GA4 recommended events
2. Generates correct dataLayer push code
3. Provides GTM variable setup instructions

### Common Workflows

#### Workflow 1: E-commerce Setup

When implementing GA4 e-commerce tracking.

**Steps:**
1. Generate view_item event
2. Generate add_to_cart event
3. Generate purchase event

## Best Practices

### Do's ✅

- Use GA4 recommended event names when possible
- Keep parameter names lowercase with underscores
- Pass consistent currency and value parameters

### Don'ts ❌

- Send PII (Personally Identifiable Information) in events
- Create duplicate events for the same action

## Security & Safety

### Data Handling

- **PII:** Strictly prohibited from being included in any dataLayer pushes generated by this skill.

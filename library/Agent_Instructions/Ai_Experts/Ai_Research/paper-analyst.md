# AI Research Paper Analyst

## Role
AI research specialist focused on reading, analyzing, summarizing, and extracting insights from academic papers in machine learning and AI.

## Core Competencies
- Paper comprehension and summarization
- Critical analysis and evaluation
- Related work synthesis
- Methodology assessment
- Results interpretation
- Research trend identification

## Paper Structure Understanding
### Standard Sections
1. **Abstract**: High-level summary
2. **Introduction**: Problem, motivation, contributions
3. **Related Work**: Prior research, positioning
4. **Methodology**: Technical approach
5. **Experiments**: Setup, datasets, baselines
6. **Results**: Findings and analysis
7. **Discussion**: Limitations, implications
8. **Conclusion**: Summary and future work

## Reading Strategies
### Three-Pass Approach
**First Pass (5-10 min)**
- Read title, abstract, introduction, conclusion
- Skim section headers and figures
- Decide if worth deeper read

**Second Pass (1 hour)**
- Read carefully, but skip proofs/details
- Note key points and figures
- Identify contributions and novelty

**Third Pass (4-5 hours)**
- Deep understanding
- Reproduce logic and proofs
- Critique methodology
- Identify weaknesses

### Targeted Reading
- Skip to sections relevant to your needs
- Focus on methodology for implementation
- Results for sota comparisons
- Related work for literature review

## Critical Analysis
### Evaluate Contributions
- **Novelty**: Truly new or incremental?
- **Significance**: Important problem?
- **Impact**: Practical or theoretical value?
- **Rigor**: Sound methodology?

### Assess Methodology
- Appropriate baselines?
- Sufficient experiments?
- Statistical significance?
- Ablation studies?
- Reproducibility (code available?)

### Check Results
- Consistent with claims?
- Fair comparisons?
- Cherry-picking avoided?
- Error bars and confidence intervals?
- Generalization beyond specific datasets?

## Summarization Framework
### TLDR (1-2 sentences)
- Core contribution in plain language
- Main result or finding

### Key Points
- Problem addressed
- Proposed solution/method
- Main experiments
- Key results
- Limitations

### Technical Details
- Architecture/algorithm specifics
- Training procedure
- Hyperparameters
- Datasets and evaluation metrics

### Takeaways
- What works and why
- When to use this approach
- Open questions
- Follow-up opportunities

## Reproducibility
### What to Look For
- Code availability (GitHub)
- Dataset accessibility
- Hyperparameters specified
- Random seed reporting
- Computational requirements
- Preprocessing details

### Red Flags
- No code or data
- Missing implementation details
- Vague experimental setup
- Single runs (no error bars)
- Incomparable baselines

## Staying Current
### Strategies
- Daily arXiv digests (ML Subreddit, Import AI)
- Follow key researchers on Twitter/X
- Subscribe to newsletters (TLDR AI, The Batch)
- Attend virtual talks/seminars
- Paper reading groups

### Time Management
- Skim many, read few deeply
- Prioritize by relevance and venue
- Use Twitter threads for quick summaries
- Save for later with tags

## Common Pitfalls
- Taking claims at face value
- Ignoring limitations section
- Not checking baselines
- Skipping related work
- Missing statistical significance
- Accepting "state-of-the-art" without context

*Read critically, summarize clearly, build knowledge.* 📚

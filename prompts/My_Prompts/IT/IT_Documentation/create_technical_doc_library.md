# IT Technical Documentation Library Generator Prompt

## ROLE

You are a senior IT documentation architect, enterprise architect, and
cyber security governance expert.

Your task is to design and generate a comprehensive, structured,
enterprise-grade technical documentation library suitable for a
mid-to-large IT organization.

The documentation must follow industry best practices including (where
appropriate):

-   ISO 27001 / 27002
-   NIST CSF / NIST 800-series
-   COBIT
-   ITIL
-   TOGAF
-   CIS Controls
-   Zero Trust Architecture principles

All documentation must be written in professional, implementation-ready
language suitable for internal enterprise use.

------------------------------------------------------------------------

## OBJECTIVE

Build a complete structured documentation library across the following
domains:

### Core Domains

1.  Identity & Access Management (IAM)
2.  Cyber Security
3.  IT Architecture
4.  IT Infrastructure
5.  IT Development / Software Engineering
6.  IT Operations
7.  Governance, Risk & Compliance (GRC)

### Additional Required Domains

Also include any other relevant IT documentation domains typically
required in an enterprise environment, such as:

-   Cloud Security & Architecture
-   DevSecOps
-   Data Governance
-   Business Continuity & Disaster Recovery
-   Security Operations (SOC)
-   Vulnerability Management
-   Incident Response
-   Change Management
-   Asset Management
-   Endpoint Management
-   Network Security
-   Logging & Monitoring
-   Vendor Risk Management
-   Privacy & Data Protection
-   AI Governance (if applicable)

------------------------------------------------------------------------

### Phase 2 -- Document Templates

For each document type, create a reusable template including:

-   Document Purpose
-   Scope
-   Audience
-   Definitions
-   Roles & Responsibilities
-   Control Objectives
-   Technical Requirements
-   Implementation Guidelines
-   Process Flow
-   Exception Handling
-   Audit & Compliance Mapping
-   Metrics / KPIs
-   Review Cycle

Templates must be detailed enough to be implementation-ready.

------------------------------------------------------------------------

### Phase 3 -- Domain-Specific Documentation Library

For EACH domain:

1.  List all required documentation artifacts
2.  Briefly describe the purpose of each artifact
3.  Identify ownership (e.g., CISO, IAM Lead, DevOps Lead,
    Infrastructure Manager)
4.  Map to relevant frameworks (NIST, ISO, CIS, etc.)
5.  Assign priority level (Critical / High / Medium / Low)
6.  Assign maturity level (Foundational / Intermediate / Advanced)

------------------------------------------------------------------------

### Phase 4 -- IAM Deep Dive (Advanced)

Provide expanded documentation artifacts specifically for:

-   Identity Lifecycle Management
-   Privileged Access Management
-   Role-Based Access Control (RBAC)
-   Attribute-Based Access Control (ABAC)
-   Federation & SSO
-   MFA & Passwordless
-   Conditional Access
-   Zero Trust Identity
-   Access Reviews & Certifications
-   JML (Joiner-Mover-Leaver) Process
-   Service Accounts
-   Secrets Management
-   API Access Security
-   Cloud IAM
-   Third-Party Access

For each, generate:

-   Architecture overview
-   Control objectives
-   Technical components
-   Implementation considerations
-   Common failure points
-   Audit considerations
-   KPIs

------------------------------------------------------------------------

### Phase 5 -- Maturity Model

Create a 5-level maturity model for:

-   IAM
-   Cyber Security
-   DevSecOps
-   Infrastructure
-   Governance

Each level should clearly define:

-   Process maturity
-   Technology maturity
-   Automation maturity
-   Governance maturity
-   Risk posture

------------------------------------------------------------------------

### Phase 6 -- Governance Model

Provide:

-   RACI model
-   Role definitions
-   Ownership model
-   Escalation paths
-   Documentation lifecycle governance

------------------------------------------------------------------------

## STYLE REQUIREMENTS

-   Professional enterprise tone
-   Structured formatting with headings
-   Clear bulleting
-   Avoid fluff
-   Avoid vague language
-   No marketing language
-   Must be implementation-oriented
-   Assume 1,000+ employee enterprise scale

You may also instruct the AI:

-   Target industry (Finance, Healthcare, SaaS, Government, etc.)
-   Regulatory requirements (SOX, HIPAA, PCI-DSS, FedRAMP, etc.)
-   Cloud-first or hybrid
-   Zero Trust maturity level
-   Multi-cloud (AWS/Azure/GCP)
-   On-prem AD vs Entra ID vs Okta
-   DevOps maturity

### Advanced Instruction (Recommended)

Add this at the top for improved results:

> Do not generate full documents immediately. First design the
> structure. Then wait for confirmation before expanding any section.

------------------------------------------------------------------------

## Usage Recommendation

For best results, execute in stages:

1.  Generate structure only.
2.  Refine structure.
3.  Generate templates.
4.  Generate domain artifacts.
5.  Deep dive IAM separately.

This improves clarity, scalability, and output quality.

# UI Design System Creator

## Purpose
Build a complete user interface design system, including component specifications, color systems, typography rules and interaction modes.

## Instructions
You are an experienced UI/UX designer and design systems expert. Please help me create a complete and consistent UI design system framework for the following projects:

### Project information

Please provide the following information from which I will create a design system:

- Product/project type: [such as mobile applications, websites, management backends, e-commerce platforms, etc.]
- Brand identity: [brand name, core values, personality traits]
- Target platform: [such as iOS, Android, Web, cross-platform, etc.]
- Design style preference: [such as minimalism, skeuomorphism, flatness, new skeuomorphism, etc.]
- Target users: [Main user groups and their characteristics]
- Competitive products or reference cases: [favorite design system or product interface example]
- Special requirements: [Accessibility requirements, multi-language support, dark mode, etc.]

### The design system framework I will provide:

#### 1. Design principles

- Core design principles (3-5)
- Definition and practical guidance for each principle
- How to embody these principles in products

#### 2. Color system

- Main and secondary colors (specific color numbers provided)
- Functional colors (success, warning, error, information, etc.)
- Neutral color palette
- Color usage specifications and usage scenarios
- Accessibility considerations (contrast requirements)
- Dark mode variant (if required)

#### 3. Typesetting system

- Font family selection and reasons
- Text size hierarchy (title, subtitle, body, etc.)
- Line height, word spacing and paragraph spacing rules
- Guidelines for using font weight
- Responsive typography adjustment rules

#### 4. Grid and Layout

- Basic grid system definition
- Spacing and margin rules
- Responsive layout breakpoints and behaviors
- Alignment principles

#### 5. Component library framework

Provided separately for the following basic components:

- Button (type, status, size)
- Form elements (input boxes, selectors, checkboxes, etc.)
- Cards and containers
- Navigation elements (menus, tabs, etc.)
- Notifications and prompts (warnings, errors, success messages, etc.)
- Modals and dialog boxes
- Data display components (tables, lists, etc.)
- Loading state and empty state
- Icon system guide

Each component description includes:

- Appearance and structure definition
- Various status changes
- size variations
- Usage scenarios and principles

#### 6. Interactive mode

- Touch/click feedback
- State transition and animation principles
- Gesture interaction specifications
- Timing and principles of using motion effects

#### 7. Image and Icon Guidelines

- Icon style and creation principles
- Guidelines for use of illustrations
- Image processing guidelines
- Image proportions and cropping rules

#### 8. Implementation Guide

- Naming convention
- File organization suggestions
- Code implementation suggestions (such as CSS variables, component structure)
- Version control and update process

#### 9. Example demonstration

- How to apply the design system in key pages or processes

Please provide detailed, specific guidance, not just generalities. Provide a clear rationale for the selection of each element, explaining how the design system is consistent and reflects the brand personality.

## Output Format
Provide the output in a clear, well-structured format with appropriate headings and sections.

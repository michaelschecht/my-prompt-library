# 6. IT Operations Prompt Library

This document contains categorized prompts for building IT Operations documentation, standards, and runbooks.

1. Create an incident management process document following ITIL guidelines, with phases: detection, response, resolution, and post-mortem.
2. Develop a service request catalog with standard request items (password reset, software install) and fulfillment workflows.
3. Document a standard for server monitoring and alerting, including thresholds for CPU, memory, disk, and critical services.
4. Write a shift handover template for 24/7 operations teams to ensure continuity.
5. Define a standard operating procedure (SOP) for database backup and restoration verification.
6. Create a standard for application deployment to production, including change windows, communication, and rollback plans.
7. Develop a capacity management process with regular reviews of resource utilization and forecasting.
8. Write an SOP for handling user-reported issues, including initial triage, categorization, and escalation.
9. Define a standard for system health checks (daily/weekly) and reporting.
10. Create a maintenance window schedule and communication template for planned downtime.
11. Document a standard for IT asset management (ITAM) including inventory tracking, software licensing, and audits.
12. Develop a standard for service level agreement (SLA) monitoring and breach notification.
13. Write an SOP for emergency access requests (break-glass) including approval and logging requirements.
14. Define a standard for operational documentation handover from projects to operations.
15. Create a runbook template for common operational tasks (e.g., restarting services, clearing queues).
16. Develop a standard for vendor management and support escalation procedures.
17. Write an SOP for new environment provisioning (dev, test, prod) with configuration baselines.
18. Define a standard for IT operations metrics reporting (e.g., uptime, MTTR, ticket volume).
19. Create a disaster recovery exercise plan for operations teams, including scenarios and success criteria.
20. Document a standard for configuration management database (CMDB) maintenance and data quality checks.

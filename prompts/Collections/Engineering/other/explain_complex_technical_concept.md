# Prompt: Explain Complex Technical Concept (Non-Technical Audience)

## Description
This prompt provides a template for explaining a complex technical concept to a non-technical audience using an analogy.

## Template

### 1. Concept
*   **Concept Name:** API (Application Programming Interface)
*   **Audience:** Non-technical beginners.

### 2. Analogy
*   **Analogy:** An API is like a waiter in a restaurant.
*   **Explanation:**
    *   You (the user) are a customer in a restaurant.
    *   The kitchen is the system that has the data or functionality you want.
    *   You can't go directly into the kitchen and get your food. You need to ask the waiter.
    *   The waiter (the API) takes your order (the request) to the kitchen (the system).
    *   The kitchen prepares your food (the data or functionality).
    *   The waiter brings the food back to you (the response).
    *   The API acts as an intermediary that allows you to access the functionality of a system without having to understand the complexity of the system itself.

### 3. Key Takeaways
*   An API is a way for different applications to talk to each other.
*   It simplifies development by abstracting away the complexity of the underlying systems.
*   APIs are everywhere! They are used by many of the apps and websites you use every day.

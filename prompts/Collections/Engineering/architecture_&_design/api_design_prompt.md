# API Design Specification (Based on IETF RFC 9110 and OpenAPI)

**Prompt:**
"Design an API for [Service] following IETF RFC 9110 (HTTP semantics) and OpenAPI best practices.
Include:
1. Resource model and URIs.
2. Methods, status codes, and error schema.
3. Pagination, filtering, and versioning strategy.
4. Auth scheme and rate limiting.
5. Example requests/responses.

Output as a mini design doc with OpenAPI-like sections."

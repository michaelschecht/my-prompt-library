Map the structure and key patterns of an unfamiliar codebase.

## Prompt

```
Explore this codebase and produce a structured summary:
1) Top-level directory structure
2) Entry points and main modules
3) Key design patterns used
4) External dependencies
5) Anything unusual or noteworthy

Read files broadly before going deep.
```

## Usage Notes

- Run this at the start of any new project engagement
- The breadth-first approach surfaces the big picture before details
- Useful for onboarding, code reviews, or audits
- Output can seed a CLAUDE.md

# 3. IT Architecture Prompt Library

This document contains categorized prompts for building IT Architecture documentation, standards, and decision records.

1. Create an enterprise architecture principles document guiding solution design decisions (e.g., reusability, interoperability, security by design).
2. Develop a reference architecture for web applications covering load balancers, application servers, databases, and caching layers.
3. Document a standard for API gateway usage, including rate limiting, authentication, and logging requirements.
4. Write an architecture decision record (ADR) template for capturing context, trade-offs, and decisions.
5. Define a microservices architecture standard covering service discovery, resilience patterns (circuit breakers), and observability.
6. Create a data architecture guideline covering data lakes, data warehouses, and real-time streaming platforms.
7. Develop a technology lifecycle management process for evaluating, adopting, and retiring technologies.
8. Document a standard for infrastructure as code (IaC) tooling and repository structure (e.g., Terraform modules, CloudFormation).
9. Write a containerization and orchestration standard (Docker, Kubernetes) including image security, resource limits, and namespace isolation.
10. Establish a solution architecture review board (ARB) charter with governance processes and decision-making criteria.
11. Create a standard for documenting system context and container diagrams using the C4 model.
12. Define a disaster recovery architecture for critical applications, including replication strategies and failover automation.
13. Document a standard for integrating on-premises systems with cloud services via VPN, Direct Connect, or ExpressRoute.
14. Develop a high-level architecture for a global content delivery network (CDN) strategy to improve performance.
15. Write a standard for application logging and monitoring from an architecture perspective (log formats, aggregation, retention).
16. Create a reference architecture for real-time data processing using Apache Kafka or similar.
17. Define a standard for session management in distributed systems (e.g., sticky sessions vs. distributed caches).
18. Document a cloud-native design pattern guide (e.g., 12-factor app principles) for developers.
19. Develop an architecture for secure file transfer (MFT) within and outside the organization.
20. Write a standard for technology stack selection criteria for new projects (programming languages, frameworks, databases).

# 7. Cloud Architecture & Security Prompt Library

This document contains categorized prompts for building Cloud Architecture and Security documentation, standards, and controls.

1. Create a cloud adoption framework (CAF) covering governance, security, operations, and finance.
2. Develop a multi-cloud architecture strategy with principles for workload placement and provider selection.
3. Document a standard for cloud account structure (e.g., AWS Organizations, management account, workload accounts).
4. Write a cloud network architecture standard covering VPC design, subnets, transit gateways, and network segmentation.
5. Define a standard for cloud identity and access management (IAM) including roles, policies, and permission boundaries.
6. Create a standard for cloud storage (S3, blob, etc.) including encryption, access logging, and lifecycle policies.
7. Develop a standard for cloud compute (EC2, VMs) including instance families, auto-scaling, and spot/fleet usage.
8. Write a standard for cloud database services (RDS, Aurora, Cloud SQL) including backups, replication, and encryption.
9. Define a standard for cloud security posture management (CSPM) tooling and continuous compliance monitoring.
10. Create a standard for container orchestration (EKS, AKS, GKE) including security groups, pod identity, and network policies.
11. Develop a standard for serverless computing (Lambda, Functions) including timeout, memory, and security best practices.
12. Write a standard for cloud cost management and optimization (FinOps) including tagging, budgets, and rightsizing.
13. Define a standard for cloud backup and disaster recovery across regions and providers.
14. Create a standard for cloud API gateway configuration, including throttling, authentication, and logging.
15. Develop a standard for cloud logging and monitoring (CloudTrail, CloudWatch, Azure Monitor) with retention and alerting.
16. Write a standard for cloud infrastructure as code (Terraform, CloudFormation) with module design and state management.
17. Define a standard for cloud workload vulnerability scanning and remediation.
18. Create a standard for cloud service catalog and self-service provisioning for developers.
19. Develop a standard for cloud compliance frameworks (CIS, NIST, PCI) mapping to cloud-native controls.
20. Write a standard for cloud incident response, including forensic data collection and isolation procedures.

# 5. IT Development / DevSecOps Prompt Library

This document contains categorized prompts for building IT Development and DevSecOps documentation, standards, and pipelines.

1. Create a development environment setup guide for new hires, including tools, repositories, and access.
2. Document a branching strategy (e.g., GitFlow, trunk-based development) with naming conventions and merge requirements.
3. Develop a code review standard covering mandatory checks, reviewer assignments, and approval criteria.
4. Write a continuous integration (CI) pipeline standard with stages for build, unit tests, and static code analysis.
5. Define a continuous delivery (CD) pipeline standard for deploying to various environments (dev, test, prod).
6. Create a standard for container image builds, including base image selection, vulnerability scanning, and tagging.
7. Document a release management process covering versioning, changelog generation, and rollback procedures.
8. Develop a standard for secret management in CI/CD pipelines (e.g., HashiCorp Vault, AWS Secrets Manager).
9. Write a standard for infrastructure as code (IaC) testing using tools like Terratest or Checkov.
10. Define a developer onboarding checklist for access to source control, CI/CD tools, and cloud environments.
11. Create a standard for API development including OpenAPI/Swagger specifications, versioning, and documentation.
12. Document a standard for dependency management (e.g., npm, pip, Maven) including license compliance and vulnerability checks.
13. Develop a chaos engineering practice document outlining principles, game days, and success metrics.
14. Write a standard for logging and structured logging from applications (e.g., JSON format, correlation IDs).
15. Create a standard for application performance monitoring (APM) agent configuration and custom metric instrumentation.
16. Define a standard for database migration scripts and automation within CI/CD pipelines.
17. Document a process for managing technical debt, including tracking, prioritization, and remediation.
18. Develop a standard for feature flags/toggles, including lifecycle management and removal.
19. Write a standard for securing CI/CD pipelines (e.g., access controls, audit logging, artifact signing).
20. Create a developer productivity metrics framework (e.g., lead time, deployment frequency, mean time to recover).

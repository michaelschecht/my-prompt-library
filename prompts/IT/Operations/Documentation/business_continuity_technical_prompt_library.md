# 12. Business Continuity (Technical Aspects) Prompt Library

This document contains categorized prompts for building technical Business Continuity and Disaster Recovery documentation.

1. Create a technical disaster recovery policy defining RTO/RPO objectives for critical applications and systems.
2. Develop a business impact analysis (BIA) template for technical services, including dependencies and impact criteria.
3. Write a disaster recovery runbook template for a typical 3-tier application (web, app, database).
4. Define a standard for backup and restore procedures, including backup types (full, incremental) and validation testing.
5. Create a standard for failover and failback procedures for critical infrastructure (servers, networks, storage).
6. Develop a standard for data replication strategies (synchronous, asynchronous) for databases and file systems.
7. Write a standard for disaster recovery testing, including tabletop, simulation, and full failover exercises.
8. Define a standard for recovery site requirements (hot, warm, cold) and infrastructure readiness.
9. Create a standard for cloud-based disaster recovery (e.g., AWS Elastic Disaster Recovery, Azure Site Recovery).
10. Develop a standard for communication and escalation during a disaster event.
11. Write a standard for technical recovery documentation (system configurations, network diagrams, contact lists).
12. Define a standard for dependency mapping and critical path analysis for recovery.
13. Create a standard for post-disaster recovery validation and return-to-normal operations.
14. Develop a standard for disaster recovery metrics (e.g., recovery time, recovery point, test success rate).
15. Write a standard for cyber recovery (air-gapped backups, immutable storage) to protect against ransomware.
16. Define a standard for third-party disaster recovery coordination (cloud providers, vendors).
17. Create a standard for disaster recovery plan maintenance and version control.
18. Develop a standard for training and awareness for technical teams on disaster recovery procedures.
19. Write a standard for integrating business continuity with change management (ensuring changes don't break recovery).
20. Define a standard for continuous improvement of disaster recovery capabilities based on tests and incidents.

# 13. Change & Release Management Prompt Library

This document contains categorized prompts for building Change and Release Management documentation, standards, and processes.

1. Create a change management policy defining change types (standard, normal, emergency) and approval paths.
2. Develop a standard change request template with fields for risk assessment, implementation plan, and rollback plan.
3. Write a standard for change advisory board (CAB) meetings, including agenda, quorum, and decision-making.
4. Define a standard for emergency change procedures, including expedited approval and post-change review.
5. Create a standard for change scheduling and conflict management (change windows, freezes).
6. Develop a standard for change impact assessment, including customer communication and stakeholder notification.
7. Write a standard for change documentation requirements (technical details, test results, back-out steps).
8. Define a standard for change success metrics and failure analysis (e.g., change success rate, failed changes).
9. Create a standard for release management, including versioning, packaging, and deployment coordination.
10. Develop a standard for release notes and communication to users and support teams.
11. Write a standard for integrating change management with IT service management (ITSM) tools.
12. Define a standard for change approval workflows based on risk and impact (e.g., single approver vs. CAB).
13. Create a standard for post-implementation review (PIR) of major changes.
14. Develop a standard for change freeze periods (e.g., holiday seasons, peak business times).
15. Write a standard for managing changes to critical systems (e.g., core banking, healthcare).
16. Define a standard for change management audit trail and compliance reporting.
17. Create a standard for automated change deployment via CI/CD pipelines with approval gates.
18. Develop a standard for change management training and awareness for all staff.
19. Write a standard for managing configuration items (CIs) affected by changes.
20. Define a standard for continuous improvement of change and release management processes.

# 14. Asset & Configuration Management Prompt Library

This document contains categorized prompts for building Asset and Configuration Management documentation, standards, and processes.

1. Create an asset management policy covering hardware, software, and virtual assets lifecycle.
2. Develop a standard for configuration management database (CMDB) structure and data model.
3. Write a standard for asset discovery and inventory (agents, network scans, cloud APIs).
4. Define a standard for asset classification (e.g., critical, non-critical) based on business impact.
5. Create a standard for software license management, including compliance tracking and optimization.
6. Develop a standard for hardware asset tracking (procurement, deployment, maintenance, disposal).
7. Write a standard for configuration item (CI) attributes and relationships (e.g., runs on, connects to).
8. Define a standard for configuration baselines and drift detection.
9. Create a standard for asset tagging and labeling (physical and virtual).
10. Develop a standard for asset lifecycle state management (ordered, in stock, deployed, retired).
11. Write a standard for configuration change control and audit trails.
12. Define a standard for integrating CMDB with ITSM tools (e.g., ServiceNow) for incident, problem, and change management.
13. Create a standard for cloud asset management across multiple providers.
14. Develop a standard for asset reconciliation and data quality checks (e.g., missing CIs, orphaned assets).
15. Write a standard for asset valuation and financial reporting (depreciation, capitalization).
16. Define a standard for mobile device and endpoint asset management (MDM/EMM).
17. Create a standard for asset retirement and secure disposal (data wiping, physical destruction).
18. Develop a standard for configuration management for network devices and infrastructure.
19. Write a standard for asset management reporting and dashboards for stakeholders.
20. Define a standard for continuous improvement of asset and configuration management processes.
